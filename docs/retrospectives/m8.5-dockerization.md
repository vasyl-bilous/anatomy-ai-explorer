# Retrospective: M8.5 — Dockerization

_Date: 2026-05-31 · Status: done_

## What shipped

- A **multi-stage `Dockerfile`** (deps → build → lean runtime) that builds the
  React client and the NestJS API into one `node:22-slim` image.
- `docker-entrypoint.sh` — the release phase: `prisma migrate deploy` →
  `prisma:seed:prod` (idempotent) → `node dist/main.js`.
- `docker-compose.yml` — Postgres (with healthcheck) + the API
  (`depends_on: service_healthy`); `docker compose up --build` runs the whole stack
  on a single origin. `.dockerignore` added.
- Verified live in Docker: SPA, deep-link fallback, regions (10 body / 7 brain
  seeded), health ready, POST analyses → 202.

## Deviations from the plan

- Docker was originally out of scope (the ADR called it a "more time" item). Pulled
  it in to get a one-command local run and a deployable Render image.

## Key decisions (top 3–5)

1. **`node:22-slim`, not alpine** — Prisma's `engineType="binary"` needs glibc;
   alpine (musl) would require `binaryTargets` + `libc6-compat` and is a classic
   "works locally, breaks in Docker" trap. Reliability over ~70 MB for a demo image.
2. **Seed from the entrypoint, not app runtime** — consistent with ADR 0001: the
   container's release phase migrates + seeds (idempotent, so restart-safe) before
   the server starts; the app itself never seeds.
3. **Compiled seed in prod (`node dist/.../seed.js`)** — the dev seed runs via
   `tsx`, a dev dependency absent from the `--omit=dev` runtime image. Added
   `prisma:seed:prod`/`db:setup:prod` that run the build output.
4. **Single origin in the container too** — the same ServeStaticModule setup serves
   SPA + API on `:4000`; compose only needs to expose one port.

## Lessons learned

- Three build-time traps, each only visible when actually building/running the
  image (not from `npm run build`):
  - `npm ci` triggers the `prepare` husky hook → fails with no `.git` →
    `--ignore-scripts`.
  - `prisma generate` reads `prisma.config.ts`, which resolves `DATABASE_URL` →
    pass a dummy build-time `DATABASE_URL` (generate doesn't connect).
  - `tsx` isn't in the prod image → seed must run the compiled JS.
- Compose merges list fields (like `ports`) by **appending**, not replacing — a
  port override needs the `!override` YAML tag, or both ports end up published.
  (Came up while testing alongside an existing local Postgres on 5432.)
