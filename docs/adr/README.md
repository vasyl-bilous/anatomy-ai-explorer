# Architecture Decision Records

ADRs capture architecturally meaningful decisions that will outlive the milestone they were made in — a pattern choice, a technology rejection, a non-trivial trade-off, or any decision a future engineer would otherwise re-litigate.

## Conventions

- One ADR per decision. File name: `NNNN-kebab-slug.md` (zero-padded, sequential), e.g. `0001-state-management-approach.md`.
- Use [`template.md`](./template.md) for the structure: **Status · Decision · Context · Consequences**.
- ADRs are **immutable once `Accepted`**. If a decision changes, write a new ADR and mark the old one `Superseded by ADR-XXXX`.
- Per-milestone retrospectives live in [`../retrospectives/`](../retrospectives/) and link to the relevant ADRs instead of duplicating rationale.
